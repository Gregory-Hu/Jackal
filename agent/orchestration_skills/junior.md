# Orchestration Skills: 初级工程师

## 技能概述

初级工程师的编排技能相对简化，重点关注：
- 理解任务
- 执行任务
- 报告状态

---

## 技能列表

### 1. Task Understanding (任务理解)

**技能 ID**: `orch.understand`

**描述**: 理解分配的任务

**子技能**:
| 子技能 | 描述 |
|--------|------|
| `read_task` | 阅读任务 |
| `ask_clarify` | 澄清疑问 |

**输入**:
- 任务描述
- 相关文档链接

**输出**:
- 任务理解确认
- 问题列表（如有）

**流程**:
```
1. 阅读任务描述
2. 检查相关资源
3. 列出疑问
4. 寻求澄清（如有需要）
5. 确认理解
```

---

### 2. Simple Planning (简单规划)

**技能 ID**: `orch.simple_plan`

**描述**: 为简单任务制定计划

**子技能**:
| 子技能 | 描述 |
|--------|------|
| `list_steps` | 列出步骤 |
| `estimate_time` | 估算时间 |

**计划模板**:
```markdown
## 任务计划

### 步骤
1. [ ] 步骤 1
2. [ ] 步骤 2
3. [ ] 步骤 3

### 预计时间
- 总时间：X 小时
- 步骤 1: X 小时
- 步骤 2: X 小时
```

---

### 3. Status Reporting (状态报告)

**技能 ID**: `orch.report_status`

**描述**: 报告任务状态

**子技能**:
| 子技能 | 描述 |
|--------|------|
| `update_progress` | 更新进度 |
| `report_blocker` | 报告阻塞 |
| `request_help` | 请求帮助 |

**状态更新格式**:
```markdown
**任务**: [任务 ID]
**状态**: 进行中 (50%)
**当前**: 正在做 XX
**下一步**: 将要做 XX
**阻塞**: 无 / [阻塞内容]
```

---

### 4. Work Logging (工作记录)

**技能 ID**: `orch.log_work`

**描述**: 记录工作内容

**子技能**:
| 子技能 | 描述 |
|--------|------|
| `log_time` | 记录时间 |
| `log_decision` | 记录决策 |
| `log_learning` | 记录学习 |

**日志格式**:
```markdown
## 工作日志 - [日期]

### 时间
- 09:00-10:30: 任务 A
- 10:30-12:00: 任务 B

### 决策
- [决策内容] + 原因

### 学习
- [学到的东西]
```

## 完整工作流

```
┌─────────────────┐
│  Task           │  接收任务
│  Understanding  │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Simple         │  简单规划
│  Planning       │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Execution      │  执行 (使用 Duty Skills)
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Status         │  状态报告
│  Reporting      │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Work           │  工作记录
│  Logging        │
└─────────────────┘
```

---

## 与 Duty Skills 的配合

```
Orchestration Skills          Duty Skills
─────────────────           ────────────────
orch.understand             → 理解要做什么
orch.simple_plan            → 列出步骤
                              ↓
                            duty.task_exec   (执行任务)
                            duty.basic_coding(编码)
                            duty.testing     (测试)
                              ↓
orch.report_status          → 报告状态
orch.log_work               → 记录工作
```

## 上报机制

### 何时上报

| 情况 | 上报 |
|------|------|
| 需求不明确 | 是 |
| 技术难题超过 1 小时 | 是 |
| 需要权限 | 是 |
| 进度延迟 > 20% | 是 |

### 上报格式

```markdown
**上报类型**: [技术难题/需求澄清/权限申请/进度延迟]

**问题描述**:
[详细描述]

**已尝试**:
- [尝试 1]
- [尝试 2]

**需要**:
[需要的帮助]
```
